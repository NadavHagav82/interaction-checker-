  <behavior>
    **parseImperialFraction(input: string): number**

    Accepted formats (5 required):
    - "1 3/4" -> 1.75 (space-separated mixed number)
    - "1-3/4" -> 1.75 (hyphen-separated mixed number)
    - "3/4" -> 0.75 (fraction only)
    - "0.75" -> 0.75 (decimal)
    - "3" -> 3.0 (integer)

    Whitespace tolerance:
    - "  1 3/4  " -> 1.75 (leading/trailing whitespace trimmed)
    - "1  3/4" -> 1.75 (extra internal whitespace)

    Rejection cases (must throw ParseError):
    - "" (empty string)
    - "abc" (non-numeric)
    - "1/0" (zero denominator)
    - "6 3/4 inches" (trailing text)
    - "1 2 3" (multiple numbers without fraction)

    **isValidMeasurement(value: number): boolean**
    - 3.5 -> true
    - 0.25 -> true
    - 0 -> false
    - -1 -> false
    - NaN -> false
    - Infinity -> false

    **ParseError**
    - Custom error class with descriptive message
    - name property equals 'ParseError'
    - message includes the invalid input string
  </behavior>
  <implementation>
    **RED phase (tests first):**

    1. Create `inputParser.test.ts` with test cases:
       - describe('parseImperialFraction'): all 5 accepted formats with exact expected values
       - describe('whitespace handling'): leading/trailing, extra internal spaces
       - describe('rejection cases'): empty, non-numeric, zero denominator, trailing text, multiple numbers
       - describe('ParseError'): name is 'ParseError', message includes input string

    2. Run tests: ALL must FAIL (RED). Commit: `test(01-03): add failing tests for inputParser`

    **GREEN phase (implementation):**

    3. Implement `inputParser.ts`:
       - ParseError class extending Error with name='ParseError' and descriptive message
       - Three anchored regex patterns (per RESEARCH.md Pattern 3):
         - MIXED_NUMBER_RE: /^\s*(\d+)\s*[-\s]\s*(\d+)\s*\/\s*(\d+)\s*$/ for "1 3/4" and "1-3/4"
         - FRACTION_ONLY_RE: /^\s*(\d+)\s*\/\s*(\d+)\s*$/ for "3/4"
         - DECIMAL_RE: /^\s*(\d+\.?\d*)\s*$/ for "0.75" and "3"
       - parseImperialFraction: trim input, try each regex in order (mixed -> fraction -> decimal), throw ParseError if none match
       - Zero denominator check: if den === 0, throw ParseError
       - isValidMeasurement: return isFinite(value) && value > 0

       CRITICAL per RESEARCH.md:
       - NEVER use eval() to parse fractions (security vulnerability)
       - Always anchor regexes with ^ and $ (prevents "6 3/4 inches" from matching)
       - Use parseInt with radix 10 for integer parts
       - Use parseFloat for decimal parts

    4. Run tests: ALL must PASS (GREEN). Commit: `feat(01-03): implement inputParser`

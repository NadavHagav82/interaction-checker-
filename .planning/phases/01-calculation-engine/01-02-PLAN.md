  <behavior>
    **Calculator (calculator.ts):**

    suggestPinCount(boardWidth):
    - boardWidth=1.5 -> 1 (very narrow, 1 tail)
    - boardWidth=2.5 -> 2 (2-3 inch range)
    - boardWidth=6 -> 3 (6+ inch, ~1 per 2 inches)
    - boardWidth=12 -> 6

    calculateDovetail({ boardWidth:6, boardThickness:0.75, ratio:8, pinCount:3 }):
    - pinCount: 3
    - tailDepth: 0.75 (equals boardThickness)
    - angle: Math.atan(1/8) = 0.12435... radians
    - pinWidthWarning: false (6 inch board with 3 tails has sufficient pin width)
    - Sum constraint: 2*halfPinWidth_face + 3*tailWidth + 2*pinWidth_face = 6.0
      (where halfPinWidth_face = pinWidth_face/2, using 3:2 tail-to-pin ratio)
    - pinWidth (narrowest point) = pinWidth_face - 2*tailDepth*tan(angle) > 0 and > 3/16"

    calculateDovetail({ boardWidth:1.5, boardThickness:0.75, ratio:8, pinCount:3 }):
    - pinWidthWarning: true (narrow board, too many pins)
    - OR pinWidth may go negative (engine should handle gracefully)

    calculateDovetail({ boardWidth:6, boardThickness:0.75, ratio:6 }):
    - Uses hardwood angle: Math.atan(1/6) = 0.16514... radians
    - Narrower pins at narrowest point (steeper angle removes more material)

    **Formatter (formatter.ts):**

    snapTo32nds(0.4375) -> 0.4375 (already on grid)
    snapTo32nds(0.43750000000000006) -> 0.4375 (float noise removed)
    snapTo32nds(0.1) -> 0.09375 (3/32, nearest 32nd)

    toFraction32(0.4375) -> '7/16"'
    toFraction32(1.75) -> '1 3/4"'
    toFraction32(3.0) -> '3"'
    toFraction32(0.03125) -> '1/32"'
    toFraction32(0.5) -> '1/2"'
    toFraction32(0.43750000000000006) -> '7/16"' (float noise handled)

    verifySum([values from 6" board layout], 6.0) -> 0 (no drift in 32nds)
  </behavior>
